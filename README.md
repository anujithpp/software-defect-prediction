# Software Defect Prediction

A binary classification mini-project that predicts whether a software module contains a bug using historical software metrics from the **NASA PROMISE Repository (jm1)**.

---

## 📁 Project Structure

```
software-defect-prediction/
├── data/
│   └── jm1.csv                  # Dataset — auto-generated by scripts/download_dataset.py
├── src/
│   └── defect_prediction.py     # Main ML pipeline (load → preprocess → train → evaluate)
├── scripts/
│   └── download_dataset.py      # One-time setup: fetches jm1.csv from OpenML
├── requirements.txt             # Python dependencies
├── .gitignore
├── LICENSE
└── README.md
```

---

## ⚙️ Setup

### 1. Install dependencies
```bash
pip install -r requirements.txt
```

### 2. Download the dataset *(first time only)*
```bash
python scripts/download_dataset.py
```
This fetches the jm1 ARFF file from [OpenML](https://www.openml.org/d/1053), converts it to CSV, and saves it to `data/jm1.csv`.

### 3. Run the full pipeline
```bash
python src/defect_prediction.py
```

---

## 🔬 Methodology

| Step | Detail |
|------|--------|
| **Dataset** | NASA PROMISE — jm1 (10,885 modules, 21 features) |
| **Target** | `defects` column — True (buggy) / False (clean) |
| **Preprocessing** | Mean imputation → StandardScaler normalisation |
| **Split** | 80% train / 20% test (stratified) |
| **Models** | Random Forest · Gaussian Naive Bayes · Logistic Regression |
| **Metrics** | Accuracy, Precision, Recall, F1-Score, Confusion Matrix |

---

## 📊 Results

| Rank | Model | Accuracy | Precision | Recall | F1-Score |
|------|-------|----------|-----------|--------|----------|
| 🥇 | **Random Forest** | **80.98%** | 51.78% | 24.23% | **33.01%** |
| 🥈 | Gaussian Naive Bayes | 80.20% | 47.60% | 23.52% | 31.48% |
| 🥉 | Logistic Regression | 80.98% | 53.54% | 12.59% | 20.38% |

> **Note:** Low Recall across all models is expected — the dataset is class-imbalanced (~19 % defective).  
> **Random Forest** ranks first by F1-Score, the most meaningful metric under class imbalance.  
> Logistic Regression achieves the highest Precision but the lowest Recall (most conservative).
